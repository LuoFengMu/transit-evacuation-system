# 版本路线详解

> 本文档是《工程实施与版本管理方案》第 5 节的扩展，提供每个版本的详细模块清单、接口定义和版本间依赖关系。

---

## 版本依赖链

```text
v0.1.0 最小原型 + 基础地图
    │ 交付：路网加载、事件设置、需求点生成、最短路径、Streamlit 地图展示
    ↓
v0.2.0 公交调度 + 简化仿真
    │ 新增：OR-Tools CVRP、简化离散事件仿真、车辆/depot 管理
    ↓
v0.3.0 SUMO 动态仿真
    │ 新增：SUMO 路网转换、TraCI 控制、FastAPI 后端、动态拥堵记录
    ↓
v0.4.0 公交-轨道协同
    │ 新增：轨道站点容量、线路运能、接驳分流、方案对比
    ↓
v0.5.0 完整 Web 可视化
    │ 新增：交互式场景配置、仿真看板、指标仪表板、报告导出
    ↓
v1.0.0 论文答辩稳定版
    交付：固化的代码/数据/场景/环境 + 完整文档 + 对比实验结果
```

## v0.1.0 — 最小原型 + 基础地图

### 模块清单

| 模块 | 路径 | 职责 |
| --- | --- | --- |
| 路网加载 | `src/network/osm_loader.py` | 从 OSM 获取指定区域路网，构建 NetworkX 图 |
| 需求点生成 | `src/demand/generator.py` | 基于事件区域生成/导入疏散需求点 |
| 避难点管理 | `src/demand/shelter.py` | 加载避难点数据 |
| 事件定义 | `src/network/event.py` | 定义突发事件区域和受影响道路 |
| 路径计算 | `src/network/pathfinder.py` | 基于 NetworkX 计算最短路径 |
| 地图展示 | `src/visualization/map_view.py` | Streamlit 地图组件，展示路网、需求点、路径 |
| 应用入口 | `streamlit_app.py` | Streamlit 主页面 |

### Python 依赖

```text
osmnx, networkx, geopandas, shapely, streamlit, folium (或 streamlit-folium)
```

### 数据依赖

- OSM 路网（在线获取或本地 .graphml）
- 手动定义的 demand_points.geojson
- 手动定义的 shelters.geojson

### 接口（内部）

```python
# src/network/osm_loader.py
def load_network(place_name: str) -> nx.MultiDiGraph: ...
def load_network_from_file(path: str) -> nx.MultiDiGraph: ...

# src/network/pathfinder.py
def shortest_path(G, origin: Point, destination: Point) -> PathResult: ...
def compute_od_matrix(G, origins: list[Point], destinations: list[Point]) -> np.ndarray: ...
```

### 输出

- `streamlit_app.py` 运行后，浏览器可访问地图页面
- 页面显示：路网底图、事件影响区域（圆形/多边形）、需求点标记、避难点标记、最短路径线
- 点击需求点可查看：到最近避难点的距离和预计时间

## v0.2.0 — 公交调度 + 简化仿真

### 新增模块

| 模块 | 路径 | 职责 |
| --- | --- | --- |
| 车辆管理 | `src/dispatch/vehicle.py` | 加载公交车辆和 depot 数据 |
| 成本矩阵 | `src/dispatch/cost_matrix.py` | 基于路网构建距离/时间成本矩阵 |
| OR-Tools 建模 | `src/dispatch/solver.py` | CVRP/VRPTW 建模与求解 |
| 简化仿真 | `src/simulation/simple_sim.py` | 离散事件仿真：排队 + 路段通行 + 容量约束 |
| 调度展示 | `src/visualization/dispatch_view.py` | Streamlit 页面显示调度结果 |

### 简化仿真引擎设计

```text
输入：车辆路径方案 + 需求点信息 + 路段通行时间
模型：
  - 公交按路径顺序访问各需求点
  - 每个需求点排队人数按车辆容量分批上车
  - 路段通行时间 = 静态估计（本版本）或简单考虑车辆数的拥堵因子
  - 目的地容量约束检查
输出：
  - 每辆车到达每个站点的时间
  - 每批次上下车人数
  - 总疏散完成时间
  - 车辆利用率时序
```

该引擎在 v0.3.0 被 SUMO 替代，但其输出的**时间估计可作为 SUMO 仿真结果的对照基线**。

### OR-Tools 模型接口

```python
# src/dispatch/solver.py
def build_cvrp_model(
    demand_points: list[DemandPoint],
    vehicles: list[BusVehicle],
    cost_matrix: np.ndarray,
    destinations: list[Destination],
) -> DispatchResult: ...

class DispatchResult:
    vehicle_routes: dict  # vehicle_id -> [stop_sequence]
    assignments: dict     # demand_id -> vehicle_id
    total_cost: float
    unserved_demand: list
```

### 输出

- 车辆路径表（CSV）
- 调度指标汇总（总疏散时间、利用率、未服务需求）
- Streamlit 页面新增"调度结果"tab

## v0.3.0 — SUMO 动态仿真

### 新增模块

| 模块 | 路径 | 职责 |
| --- | --- | --- |
| OSM→SUMO 转换 | `scripts/osm_to_sumo.py` | 将 OSM 路网转为 SUMO .net.xml |
| Route 生成 | `src/simulation/route_builder.py` | 将调度路径转为 SUMO .rou.xml |
| TraCI 控制 | `src/simulation/traci_runner.py` | 启动/控制/停止 SUMO 仿真 |
| 事件注入 | `src/simulation/event_injector.py` | 仿真运行时注入道路封闭和容量下降 |
| 结果收集 | `src/simulation/result_collector.py` | 收集车辆轨迹和路段状态 |
| FastAPI 后端 | `src/api/main.py` | 提供仿真任务管理和结果查询接口 |
| 仿真展示 | `src/visualization/simulation_view.py` | Streamlit 展示仿真结果 |

### SUMO 路网转换流程

```text
OSMnx Graph (MultiDiGraph)
    → 提取节点和边
    → SUMO node XML (带坐标)
    → SUMO edge XML (带车道、限速、长度)
    → NETCONVERT → .net.xml
    → POLYCONVERT (可选，建筑和障碍物)
```

### FastAPI 接口

```text
GET    /api/scenarios                         # 列出场景
POST   /api/scenarios                         # 创建场景
GET    /api/scenarios/{id}                    # 获取场景详情
POST   /api/dispatch/run                      # 运行调度优化
GET    /api/dispatch/{id}/result              # 获取调度结果
POST   /api/simulation/run                    # 启动 SUMO 仿真
GET    /api/simulation/{id}/status            # 查询仿真状态
GET    /api/simulation/{id}/metrics           # 获取仿真指标
GET    /api/simulation/{id}/trajectory        # 获取车辆轨迹
```

### 输出

- SUMO 路网文件（.net.xml）
- Route 文件（.rou.xml）
- 车辆轨迹（trajectory.csv）
- 拥堵时序（congestion.csv）
- 仿真指标（metrics.csv）

## v0.4.0 — 公交-轨道协同

### 新增模块

| 模块 | 路径 | 职责 |
| --- | --- | --- |
| 轨道站点 | `src/rail/station.py` | 加载和管理轨道站点数据 |
| 轨道线路 | `src/rail/line.py` | 线路运能计算 |
| 协同优化 | `src/rail/cooperative_solver.py` | 考虑轨道容量约束的扩展调度模型 |
| 站点压力 | `src/evaluation/rail_pressure.py` | 轨道站点压力评估 |
| 方案对比 | `src/evaluation/comparison.py` | 公交直达 vs 协同方案对比 |

### 协同优化模型

```text
在 v0.2.0 CVRP 基础上新增约束：
  - 分配到轨道站点的累计人数 ≤ 站点剩余接纳能力
  - 单位时间分配到轨道站点的人数 ≤ 线路小时输送能力 × 分配比例
  - 需求点到轨道站点的成本 = 公交接驳时间 + 轨道出行时间
```

### 方案对比输出

| 方案 | 总疏散时间 | 完成率 | 公交利用率 | 站点压力 |
| --- | --- | --- | --- | --- |
| 公交直达避难点 | - | - | - | N/A |
| 公交接驳轨道 | - | - | - | - |
| 混合分流 | - | - | - | - |

## v1.0.0 — 论文答辩稳定版

### 固化清单

| 对象 | 固化方式 |
| --- | --- |
| 代码 | Git tag v1.0.0，签名 |
| 数据 | `data/manifest/data_manifest_v1.0.json` 锁定所有数据文件 |
| 场景 | 至少 3 个典型场景的 YAML 配置 |
| 环境 | `docker-compose.yml` + `requirements.txt` + `environment.yml` 版本号全部锁定 |
| 结果 | `experiments/v1.0_final/` 中保存所有对比实验 |

### 最低实验要求

| 实验 | 场景 | 对比维度 |
| --- | --- | --- |
| 基准方案 | 不启用公交-轨道协同 | 纯公交疏散 → 避难点 |
| 公交调度优化 | 启用 OR-Tools | 优化调度 vs 简单最近分配 |
| 公交-轨道协同 | 启用轨道接驳 | 协同 vs 纯公交，不同分流比例 |

### 论文交付物

- 3 组对比实验的完整结果（指标 + 图表 + 截图）
- README（含安装和运行说明）
- 系统设计文档
- API 文档
- Release Note
- 已知限制与简化假设说明
