> 本文档说明如何创建疏散场景、运行仿真实验、管理数据版本和对比方案。是《工程实施与版本管理方案》第 8-10 节的操作指南。

## 1. 场景创建流程

### 1.1 新建场景 YAML

在 `configs/scenarios/` 下创建文件，命名规则：`scenario_{version}_{name}.yaml`

```yaml
scenario_id: scenario_v0.4_flood_001
scenario_name: "暴雨道路封闭-公交轨道协同"
data_version: data_v0.4

event:
  type: "flood"
  center: [118.78, 32.05]
  radius_m: 1500
  blocked_roads:
    - road_1023
    - road_2048
  capacity_reduction_ratio: 0.5    # 影响范围内道路容量降至 50%

demand:
  total_people: 5000
  demand_points_file: "data/processed/demand_points_v0.4.geojson"

bus:
  num_buses: 30
  capacity_per_bus: 60

rail:
  enabled: true

simulation:
  engine: "sumo"
  duration_s: 7200
  step_length_s: 1.0
  random_seed: 42

expected:
  min_completion_rate: 0.80
```

### 1.2 场景类型模板

**道路封闭型（地震/暴雨/事故）**：
- 重点配置 `blocked_roads` 和 `capacity_reduction_ratio`
- 影响范围通过 `radius_m` 控制

**大客流型（大型活动/节假日）**：
- 重点配置 `demand.total_people` 和需求点分布
- 不需要 `blocked_roads`，但可设置初始拥堵等级

**多事件复合型**：
- 可定义多个 event 块
- 适合论文实验的复杂场景

### 1.3 场景验证

```bash
python scripts/validate_scenario.py configs/scenarios/scenario_v0.4_flood_001.yaml
```

验证项：
- YAML 格式合法
- 引用文件存在
- 数值范围合理（如 `completion_rate` 在 0-1 之间）
- 道路 ID 在路网中存在

## 2. 数据版本管理操作

### 2.1 创建数据清单

```bash
python scripts/generate_manifest.py \
  --version data_v0.4 \
  --osm data/osm/nanjing_2026_05_01.graphml \
  --gtfs data/gtfs/gtfs_mock_v0.2.zip \
  --demand data/processed/demand_points_v0.4.geojson \
  --output data/manifest/data_manifest_v0.4.json
```

生成的 `data_manifest_v0.4.json`：
```json
{
  "manifest_schema_version": "1.0",
  "data_version": "data_v0.4",
  "created": "2026-05-11",
  "files": {
    "osm_network": "data/osm/nanjing_2026_05_01.graphml",
    "gtfs": "data/gtfs/gtfs_mock_v0.2.zip",
    "demand_points": "data/processed/demand_points_v0.4.geojson",
    "...": "..."
  }
}
```

### 2.2 切换数据版本

场景配置中指定 `data_version` 字段，系统自动加载对应 manifest 解析文件路径：

```python
# 伪代码
manifest = load_manifest(f"data/manifest/data_manifest_{scenario.data_version}.json")
demand_points = load_geojson(manifest.files["demand_points"])
```

### 2.3 大型数据用 DVC

当数据文件超过 10MB 时：

```bash
dvc add data/osm/nanjing_full.graphml
git add data/osm/.gitignore data/osm/nanjing_full.graphml.dvc
git commit -m "Add full nanjing OSM data (DVC tracked)"
dvc push
```

## 3. 实验运行流程

### 3.1 创建实验目录

```bash
export EXP_ID="EXP-20260511-v0.4-bus-rail-flood"
mkdir -p experiments/$EXP_ID/figures
```

### 3.2 准备实验配置

```bash
cp configs/scenarios/scenario_v0.4_flood_001.yaml experiments/$EXP_ID/config.yaml
cp data/manifest/data_manifest_v0.4.json experiments/$EXP_ID/data_manifest.json
```

### 3.3 运行实验

```bash
python run_experiment.py \
  --config experiments/$EXP_ID/config.yaml \
  --output experiments/$EXP_ID/
```

### 3.4 记录版本关联

`run_experiment.py` 自动生成 `experiments/$EXP_ID/simulation_run.json`：

```json
{
  "run_id": "EXP-20260511-v0.4-bus-rail-flood",
  "git_tag": "v0.4.0",
  "data_manifest_version": "data_v0.4",
  "scenario_id": "scenario_v0.4_flood_001",
  "algorithm_version": "dispatch_or_tools_v0.2",
  "engine_version": "SUMO 1.19.0",
  "python_version": "3.11.8",
  "start_time": "2026-05-11T14:30:00",
  "end_time": "2026-05-11T15:45:00",
  "status": "completed"
}
```

### 3.5 验证实验结果

```bash
python scripts/validate_experiment.py experiments/$EXP_ID/
```

验证项：
- `config.yaml` 存在且合法
- `simulation_run.json` 版本关联完整
- 指标数值在合理范围
- 图表文件已生成

## 4. 方案对比实验

### 4.1 对比实验设计

针对同一场景，运行多个方案：

```bash
# 方案 A：公交直达避难点
python run_experiment.py --config configs/scenarios/scenario_bus_only.yaml \
  --output experiments/EXP-20260511-v0.4-bus-only/

# 方案 B：公交-轨道协同
python run_experiment.py --config configs/scenarios/scenario_bus_rail.yaml \
  --output experiments/EXP-20260511-v0.4-bus-rail/

# 方案 C：混合分流
python run_experiment.py --config configs/scenarios/scenario_hybrid.yaml \
  --output experiments/EXP-20260511-v0.4-hybrid/
```

### 4.2 生成对比报告

```bash
python scripts/compare_experiments.py \
  experiments/EXP-20260511-v0.4-bus-only \
  experiments/EXP-20260511-v0.4-bus-rail \
  experiments/EXP-20260511-v0.4-hybrid \
  --output outputs/reports/comparison_v0.4.md
```

输出对比表：

| 指标 | 公交直达 | 公交-轨道 | 混合分流 |
| --- | --- | --- | --- |
| 总疏散时间 (s) | 5400 | 4200 | 3900 |
| 疏散完成率 | 0.72 | 0.91 | 0.94 |
| 公交利用率 | 0.85 | 0.62 | 0.71 |
| 轨道站点压力 | N/A | 0.73 | 0.58 |

## 6. 环境准备与切换

### 6.1 初始化环境

```bash
# Conda（推荐）
conda env create -f environment.yml
conda activate transit-evacuation

# Pip
pip install -r requirements.txt
```

### 6.2 Docker 启动服务

```bash
docker compose up -d postgis    # 先启动数据库
docker compose up -d fastapi    # 后端
docker compose up -d streamlit  # 前端
```

### 6.3 版本切换

切换到特定版本的环境：

```bash
git checkout v0.4.0
conda env update -f environment.yml    # 同步该版本的依赖
docker compose up -d                   # 启动该版本的服务
```

## 7. 常见问题

**Q: SUMO 仿真无法启动？**
检查：`SUMO_HOME` 环境变量、SUMO 版本与 Python 版本兼容性、路网文件是否正确转换。

**Q: 实验结果与预期不符？**
检查：`simulation_run.json` 确认版本关联、`run.log` 查看错误、对比该版本的基线指标。

**Q: 如何回到旧版本的实验结果？**
查看目标版本的 `simulation_run.json`，检出对应 git tag，恢复对应数据版本，重新运行。
