# 技术蓝图与开发路线

## 城市大客流公交-轨道协同疏散仿真与调度优化系统

---

## 一、系统定位

面向大型活动散场大客流场景的公交-轨道-步行多方式协同疏散仿真系统。以徐州市为示范城市，基于真实路网和交通数据，利用运筹优化和交通仿真技术，评估不同疏散方案的效果。

**核心逻辑**：轨道交通承担主干大容量疏散，公交负责接驳补盲和压力分流，步行负责短距离接入，系统通过协同分配和仿真评价选择最优疏散方案。

---

## 二、技术架构

```
数据层: OSM路网 / 公交站 / 轨道站 / 轨道线路 / 需求点
        ↓
场景层: 事件地点 / 人群聚集半径 / 人数量级
        ↓
接入层: 需求点 → 公交站 / 轨道站 (步行5km/h)
        ↓
协同层: 5方式分配 (walk_self/walk_rail/bus_rail/bus_periphery/unserved)
        ↓
调度层: OR-Tools CVRP → 公交车辆路径
        ↓
仿真层: SUMO + TraCI → 公交运行 + 道路封闭
        ↓
轨道层: 站点容量(K_s) / 动态处理(Q_s) / 线路运能 / 站点压力(P_s)
        ↓
评价层: 疏散完成率 / 站点压力 / 公交利用率 / 方案对比
        ↓
可视化层: Streamlit + Plotly → 地图 / 图表 / 报告
```

### 技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| 路网 | OSMnx + NetworkX | 道路网络加载、最短路径 |
| 公交站 | Overpass API | 徐州真实公交站点 |
| 轨道 | 手动录入 + 公开资料推算 | 20站(1/2/3号线) + 容量模型 |
| 调度 | OR-Tools 9.x | CVRP 公交车辆路径优化 |
| 仿真 | SUMO 1.26 + TraCI | 动态交通仿真 + 道路封闭 |
| 可视化 | Streamlit + Plotly | Web交互界面 + 地图 |
| 地理 | GeoPandas + Shapely + pyproj | 空间数据处理 |
| 路网解析 | sumolib | SUMO net.xml 读取 + 坐标转换 |

## 七、关键技术决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 仿真引擎 | SUMO (非MATSim) | 工程化程度高，Python 接口好，适合毕设 |
| 调度求解器 | OR-Tools (非自定义) | 成熟稳定，CVRP直接可用 |
| 前端框架 | Streamlit (非React/Vue) | 快速原型，Python全栈 |
| 路网数据 | OSM (非商业地图) | 开源免费，覆盖徐州 |
| 简化vs高精路网 | simplify=False | 轨迹贴路明显改善(9%→54%丰富几何边) |
| 大模型报告 | 不使用 | 模板报告足够，增加不可控依赖 |
| 避难点 | 大客流场景移除 | 散场人群需要回家，非避难 |
| 事件类型 | 仅保留大客流 | 项目定位明确 |
