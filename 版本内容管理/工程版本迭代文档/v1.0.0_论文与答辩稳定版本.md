# v1.0.0 论文与答辩稳定版本

## 版本目标

v1.0.0 的目标是形成可复现、可展示、可写入论文的稳定系统版本。该版本不再追求新增大功能，而是固化系统环境、实验数据、典型场景、评价指标、结果图表和文档说明。

## 功能范围

- 固化完整工程代码。
- 固化数据版本。
- 固化典型疏散场景。
- 固化算法参数。
- 固化 SUMO 仿真环境。
- 固化 Web 演示流程。
- 输出论文实验所需指标、图表和截图。
- 完成 README、系统设计、版本说明和 Release Note。

## 数据需求

v1.0.0 应整理并冻结以下数据：

| 数据类别 | 版本要求 |
| --- | --- |
| 路网数据 | 明确城市范围、来源、采集时间、处理脚本和文件名 |
| 公交数据 | 明确站点、线路、车辆和 depot 数据版本 |
| 轨道数据 | 明确站点容量、线路运能、换乘关系和简化假设 |
| 需求数据 | 明确需求点生成方法、总人数、优先级和人群类型 |
| 事件数据 | 明确事件类型、影响范围、封闭道路和持续时间 |
| 实验结果 | 保存每组实验配置、运行日志、指标和图表 |

必须保存：

```text
data/manifest/data_manifest_v1.0.json
configs/scenarios/scenario_v1.0_final.yaml
configs/experiments/exp_v1.0_final.yaml
outputs/reports/report_v1.0.md

```

## 实验设计建议

论文或答辩版本建议至少包含三组实验：

| 实验 | 目的 |
| --- | --- |
| 基准方案 | 不启用公交-轨道协同，只进行普通公交疏散 |
| 公交调度优化方案 | 启用 OR-Tools 公交调度 |
| 公交-轨道协同方案 | 启用轨道站点容量和线路运能约束 |

建议评价指标：

- 总疏散时间。
- 疏散完成率。
- 未疏散人数。
- 平均等待时间。
- 公交车辆利用率。
- 轨道站点压力。
- 道路拥堵指数。

## 验收标准

- 新环境下可按 README 启动系统。
- 可复现至少一个完整疏散场景。
- 可展示从场景配置、调度优化、仿真运行到结果分析的完整流程。
- 所有实验结果能够追溯到代码、数据、场景和参数。
- 论文所需主要图表和指标已生成。
- 已知问题和简化假设已在文档中说明。

## 已知限制

- 轨道交通列车运行可能仍为运能简化模型。
- 地铁站内部疏散如未接入 JuPedSim，需要在论文中说明。
- 若未使用真实 GTFS 数据，需要说明模拟线路和站点的构造方法。

## 后续扩展方向

- 接入 JuPedSim，模拟地铁站内部人群疏散。
- 接入 OpenTripPlanner，增强公交-地铁-步行多方式路径规划。
- 接入 MATSim，扩展为城市级多智能体疏散行为仿真。
- 使用 React / Vue + deck.gl 提升前端交互能力。
