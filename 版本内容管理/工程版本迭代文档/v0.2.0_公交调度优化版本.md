# v0.2.0 公交调度优化 + 简化仿真验证

## 版本目标

v0.2.0 的目标是加入公交应急调度优化能力，并通过简化离散事件仿真验证调度方案的可行性。本版本不依赖 SUMO，但提供比纯静态路径更有参考价值的运行时间估计，为 v0.3.0 接入 SUMO 提供对照基线。

## 功能范围

- 新增公交车辆和 depot 数据
- 支持车辆容量约束
- 支持多个疏散需求点分配
- 支持多个避难点或接驳站点选择
- 使用 OR-Tools 求解 CVRP/VRPTW
- 简化离散事件仿真（排队模型 + 路段通行时间 + 容量约束）
- 输出每辆车的服务点序列、载客量、预计运行时间和车辆利用率
- 与静态最短路径估计的结果对比

## 数据需求

继承 v0.1.0 的全部数据，并新增：

| 数据类别 | 最小字段 |
| --- | --- |
| 公交车辆 | vehicle_id、depot_id、capacity、available_time |
| depot | depot_id、depot_name、lon、lat、available_vehicle_count |
| 调度优化输入 | optimization_id、scenario_id、vehicle_ids、demand_ids、destination_ids、cost_matrix |
| 调度优化输出 | assignment_result、vehicle_routes |

## 算法与模型

### OR-Tools 调度模型

建模为 CVRP（带容量约束的车辆路径问题）或 VRPTW（带时间窗）：

```text
minimize 总车辆运行时间
约束：车辆容量、需求点单次服务、depot 起点
```

### 简化离散事件仿真

```text
输入：车辆路径方案 + 需求点信息 + 路段通行时间
模型：
  - 公交按路径顺序访问各需求点
  - 需求点排队人数按车辆容量分批上车
  - 路段通行时间 = 静态估计 × 拥堵因子（基于路段车辆数）
  - 目的地容量约束检查
输出：每辆车到站时间、每批次上下车人数、总疏散完成时间、车辆利用率时序
```

该仿真结果在 v0.3.0 与 SUMO 输出对比，验证仿真模型的准确性。

## 验收标准

- 能读取公交车辆、depot、需求点和目的地数据
- 能构造距离或时间成本矩阵
- OR-Tools 能输出可行调度方案
- 简化仿真能输出近似的总疏散时间和车辆利用率
- 静态路径估计 vs 简化仿真结果可对比
- 可在 Web 页面查看车辆路径和服务序列

## 已知限制

- 简化仿真的拥堵模型较为粗糙，不反映真实交通流动力学
- 公交路线尚未进入 SUMO 动态执行
- 轨道站点可作为普通目的地处理，不模拟线路运能

## 下一版本衔接

v0.3.0 将 OR-Tools 输出的车辆路径转为 SUMO route 文件，在 SUMO 中模拟道路封闭、公交运行和动态拥堵，实现完整的高保真动态仿真。
