# v0.1.0 最小可运行原型 + 基础地图展示

## 版本目标

v0.1.0 的目标是跑通系统最小闭环：能够加载城市路网，设置突发事件区域，生成疏散需求点和避难点，计算基础疏散路径，并在 Streamlit 地图上交互式展示。本版本即开始搭建 Web 展示层，后续版本持续增强。

## 功能范围

- 获取或导入 OSM 道路网络
- 设置突发事件影响区域
- 生成若干疏散需求点
- 设置若干避难点
- 计算需求点到避难点的最短路径
- Streamlit 地图交互展示（路网、事件区域、需求点、避难点、路径）
- 输出基础疏散距离和预计通行时间

## 数据需求

最小数据：

| 数据类别 | 最小字段 |
| --- | --- |
| 道路路网 | road_id、from_node、to_node、geometry、length_m、speed_kmh、road_type |
| 路网节点 | node_id、lon、lat、geometry |
| 疏散需求点 | demand_id、demand_name、lon、lat、geometry、people_count、priority |
| 避难点 | shelter_id、shelter_name、lon、lat、geometry、capacity |
| 突发事件 | event_id、event_type、center_lon、center_lat、radius_m、start_time、affected_road_ids |

## 算法与模型

- OSMnx 获取路网
- NetworkX 计算最短路径
- 路径权重采用 `length_m / speed_m_s`

## 仿真与可视化

- 本版本不做动态交通仿真，只做静态路径展示
- Streamlit 页面包含：道路网络底图、事件影响范围、疏散需求点和避难点标记、最短路径线
- 点击需求点可查看路径距离和预计通行时间

## 验收标准

- 能成功加载目标区域路网
- Streamlit 页面能正常启动并显示地图
- 能生成并显示突发事件区域
- 能显示需求点和避难点
- 每个需求点都能计算一条可达路径
- 页面可交互查看路径的距离和预计通行时间

## 已知限制

- 不考虑公交车辆调度
- 不考虑道路拥堵动态变化
- 不考虑轨道交通站点容量
- 地图交互功能有限（后续版本逐步增强）

## 下一版本衔接

v0.2.0 应在本版本基础上加入公交车辆、depot、OR-Tools 调度优化和简化仿真验证，将静态疏散路径升级为公交接驳调度方案，并用简化离散事件仿真验证调度效果。
