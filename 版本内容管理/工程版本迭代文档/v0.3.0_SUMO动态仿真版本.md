# v0.3.0 SUMO 动态仿真版本

## 版本目标

v0.3.0 的目标是将公交调度结果接入 SUMO，实现道路封闭、公交车辆运行、交通状态变化和仿真结果输出。早期规划曾将 FastAPI 后端放在本版本，但后续路线已调整为按需服务化增强，v0.3.0 的核心验收以 SUMO/TraCI 动态仿真为准。

## 功能范围

- 将 OSM 路网转换为 SUMO 网络
- 将公交调度路径转换为 SUMO route 文件
- 支持突发事件导致的道路封闭或容量下降
- 通过 TraCI 控制仿真运行（启动/暂停/恢复/停止）
- 记录公交车辆轨迹、运行时间和到达状态
- 统计道路拥堵变化
- FastAPI 后端接口（已后移为 v0.6.0+ 按需增强，不作为本版本核心验收）
- 与 v0.2.0 简化仿真结果对比，验证模型精度

## 数据需求

继承 v0.1.0 和 v0.2.0 的全部数据，并新增：

| 数据类别 | 最小字段 |
| --- | --- |
| SUMO 路网映射 | road_id、sumo_edge_id |
| 仿真参数 | simulation_id、scenario_id、engine、start_time、duration_s、step_length_s、random_seed |
| 公交线路仿真 | vehicle_id、sumo_route_id、depart_time、route_edges |
| 仿真结果 | simulation_id、vehicle_id、arrival_time、travel_time、route_status |
| 仿真运行关联 | run_id、git_tag、data_manifest_version、scenario_id、engine_version、status |

## 关键流程

```text
OSM 路网 → OSMnx → SUMO netconvert → .net.xml
OR-Tools vehicle_routes → route_builder.py → .rou.xml
路网 + route + 事件 → SUMO config → TraCI 启动仿真
仿真过程 → result_collector.py → metrics.csv + trajectory.csv
```

## 验收标准

- OSM 路网能成功转换为 SUMO 网络
- 调度方案能生成 SUMO route 文件
- SUMO 仿真能正常启动和结束
- 能模拟至少一种道路封闭场景
- 能输出公交车辆运行时间和道路拥堵指标
- SUMO 仿真结果与 v0.2.0 简化仿真可对比
- FastAPI 接口可查询仿真状态和结果

## 已知限制

- 轨道交通仍未作为独立运能系统建模
- 公交调度方案先离线生成，仿真过程中可逐步支持动态重算
- 人群上下车和站点排队可先简化处理

## 下一版本衔接

v0.4.0 应加入轨道站点容量、轨道线路运能和公交接驳分流，使系统真正体现公交-轨道协同疏散。
