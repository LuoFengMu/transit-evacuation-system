# v0.4.0 公交-轨道协同版本

## 版本目标

v0.4.0 的目标是将轨道交通纳入疏散体系，把地铁站从普通目的地升级为具有容量约束和线路运能约束的大容量疏散节点。公交负责短距离接驳和受灾区域外运，轨道交通负责中长距离、大容量疏散。

## 功能范围

- 新增轨道站点数据。
- 新增轨道线路和运能数据。
- 支持轨道站点容量约束。
- 支持轨道线路小时输送能力约束。
- 支持公交接驳到不同轨道站点的分配。
- 支持站点压力评价。
- 支持公交直达避难点与公交接驳轨道站两种方案对比。

## 数据需求

继承 v0.1.0 到 v0.3.0 的全部数据，并新增：

| 数据类别 | 最小字段 |
| --- | --- |
| 轨道站点 | station_id、station_name、lon、lat、geometry、line_id、station_capacity |
| 轨道线路 | line_id、line_name、station_sequence、train_capacity、headway_min、direction |
| 公交站点 | stop_id、stop_name、lon、lat、geometry、nearest_road_id |
| 步行接驳 | walkway_id、from_node、to_node、geometry、length_m |

建议扩展数据：

- 站厅容量、站台容量、出入口数量和出入口通行能力。
- 当前站内人数、限流状态和关闭状态。
- 线路小时输送能力、应急发车间隔和中断状态。
- 公交站到地铁站的步行距离、换乘时间和通道容量。

## 算法与模型

轨道交通可先简化为三类约束：

```text
站点接纳能力 = 站厅容量 + 站台容量 + 出入口通行能力
线路输送能力 = 列车容量 × 发车频率
站点压力 = 进入站点人数 / 站点最大处理能力
```

调度优化模型需要同时考虑：

- 公交车辆容量。
- 道路通行时间。
- 地铁站剩余接纳能力。
- 轨道线路输送能力。
- 需求点到轨道站点的接驳成本。
- 避难点容量。

建议对比三类方案：

| 方案 | 说明 |
| --- | --- |
| 公交直达避难点 | 所有人由公交送往避难点 |
| 公交接驳轨道站 | 公交将人群送到地铁站，由轨道承担后续疏散 |
| 公交-轨道混合分流 | 一部分送避难点，一部分送轨道站 |

## 验收标准

- 能读取轨道站点和线路运能数据。
- 调度结果不超过轨道站点容量。
- 能计算站点压力和线路运能利用率。
- 能输出公交接驳轨道站方案。
- 能与公交直达方案进行指标对比。

## 已知限制

- 轨道列车运行可先简化为线路输送能力。
- 地铁站内部行人运动可暂不接入 JuPedSim。
- 轨道中断和列车折返能力可作为后续扩展。

## 下一版本衔接

v0.5.0 应优先将当前已拆出的主流程升级为可复现实验平台：提供 CLI 单场景入口、批量实验入口、实验 manifest 和结果汇总表。Streamlit 继续作为交互式演示与结果查看入口，但不再作为唯一运行方式。
