# v0.5.0 可复现实验平台版

## 版本目标

v0.5.0 的目标是把当前已经可交互运行的 Streamlit 研究原型，升级为可脚本化、可批量运行、可追溯的实验平台。Web 展示已经从 v0.1.0 起步，并在 v0.4.x 中形成完整闭环；本版本不再把“完善 Web”作为主线，而是优先服务论文实验复现和批量对比。

## 功能范围

- CLI 单场景运行入口：不打开网页也能运行实验
- 批量实验入口：支持多场景、多 seed、多轨道容量系数组合
- 实验 manifest：记录代码版本、场景、参数、seed、运行耗时和输出目录
- 结果汇总：生成 `summary.csv`，汇总完成率、轨道分担、公交分担、未服务人数、站点压力等指标
- 主流程集成测试：验证一次运行能产出完整输出文件
- Streamlit 继续作为交互式查看和演示入口，不作为唯一运行入口

## 运行形态

```text
单场景：
.venv/bin/python -m src.app.cli --scenario configs/scenarios/C_hybrid_cooperative.yaml --seed 42

批量实验：
.venv/bin/python -m src.app.batch --scenarios A B C --seeds 42 43 44 --capacity-factors 0.7 1.0 1.2
```

## 数据流

```text
CLI / Batch / Streamlit
    → src.app.pipeline.run_analysis(params)
    → outputs/runs/{run_id}/
    → outputs/experiments/{experiment_id}/summary.csv
    → manifest.json / summary.csv
```

## 目录与文件交付

```text
src/app/cli.py                      # 单场景命令行入口
src/app/batch.py                    # 批量实验入口
src/app/experiment.py               # 实验 manifest / summary 工具函数
tests/test_cli.py                   # CLI 参数与运行入口测试
tests/test_pipeline_integration.py  # 主流程输出文件集成测试
outputs/experiments/{experiment_id}/summary.csv
outputs/experiments/{experiment_id}/manifest.json
```

## 验收标准

- 单场景 CLI 能调用 `run_analysis(params)` 并输出 `run_id`、输出目录和核心指标
- 每次运行均生成 7 类文件：`config.yaml`、`scenario.yaml`、`metrics.json`、`run_meta.json`、`report.txt`、`dispatch_summary.csv`、`station_pressure.csv`
- 批量实验能运行 A/B/C 场景、多个 seed、多个容量系数，并生成统一 `summary.csv`
- `manifest.json` 能追溯代码版本、场景文件、参数快照、随机种子、运行耗时和输出目录
- 集成测试能验证输出文件完整性和关键指标字段
- Streamlit 与 CLI 使用同一个 `run_analysis()` 主流程，避免两套逻辑分叉

## 已知限制

- v0.5.0 不优先新增 FastAPI、数据库后台或多用户权限
- Docker 环境固化可后移到 v1.0，除非 SUMO/OSMnx 环境复现成为近期瓶颈
- 真实人口、公交时刻表、轨道站实测容量等外部数据接入后移到 v0.6.0+

## 下一版本衔接

v0.6.0+ 可根据论文和演示需要接入真实人口/公交/轨道数据，或按需引入 FastAPI、Docker、CI。v1.0.0 应围绕论文和答辩进行稳定化：固定数据、场景、参数、结果、截图、运行环境和 Release Note。
