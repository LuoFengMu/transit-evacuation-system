# v0.5.0 完整 Web 可视化版本

## 版本目标

v0.5.0 的目标是将前几个版本的功能整合为完整的交互式演示系统。注意：Web 基础展示从 v0.1.0 即已起步，本版本是在已有基础上增强交互能力、完善场景配置流程和结果分析看板。

## 功能范围

- 完善场景配置页面（事件参数、车辆参数、轨道接驳开关等）
- 仿真运行控制面板（启动/暂停/重置/进度显示）
- 疏散完成率实时曲线
- 公交轨迹动画回放
- 道路拥堵热力图
- 轨道站点压力仪表板
- 多方案对比页面
- 一键导出实验报告

## 页面结构

```text
streamlit_app.py
├── 侧边栏：场景选择 + 参数配置表单
├── Tab 1：地图展示（路网、事件、需求点、路径、轨迹动画）
├── Tab 2：调度结果（车辆路径表、需求点分配表、利用率）
├── Tab 3：仿真监控（运行状态、实时指标、进度条）
├── Tab 4：结果分析（疏散曲线、拥堵热力图、站点压力、方案对比）
└── Tab 5：报告导出
```

## 前后端数据流

```text
Streamlit (8501) → POST /api/dispatch/run → FastAPI (8000) → OR-Tools
Streamlit (8501) → POST /api/simulation/run → FastAPI (8000) → SUMO/TraCI
Streamlit (8501) → GET /api/simulation/{id}/metrics → FastAPI (8000) → PostGIS
```

## 验收标准

- 能通过 Web 页面选择或创建场景
- 能通过 Web 页面启动调度优化和 SUMO 仿真
- 能实时展示仿真进度和关键指标
- 能展示公交路径、站点分配、疏散曲线和拥堵热力图
- 能对比公交-轨道协同方案与纯公交方案
- 演示流程无需手动修改代码即可完成
- 可一键导出实验报告（含图表和指标）

## 已知限制

- Streamlit 适合快速演示，但复杂交互能力有限（如地图拖拽选点）
- 第一版可先不做多用户权限和数据库后台管理
- 仿真运行时间较长时，进度提示依赖轮询方式
